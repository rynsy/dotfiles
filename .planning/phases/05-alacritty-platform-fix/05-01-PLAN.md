2. Add a new step AFTER the stow loop (Step 4, after line 158) and BEFORE the shell change prompt (Step 5). Insert a platform-specific Alacritty config placement section. Use `ln -sf` (symlink, not copy) per research recommendation — this tracks changes in the dotfiles repo:
```bash
# Step 4.5: Place platform-specific Alacritty config
if [ -d "$DOTFILES_DIR/alacritty/platform" ]; then
  echo "--- Placing platform Alacritty config ---"
  mkdir -p "$HOME/.config/alacritty"
  case "$OS" in
    Linux)
      ln -sf "$DOTFILES_DIR/alacritty/platform/linux.toml" "$HOME/.config/alacritty/linux.toml"
      echo "  linked: linux.toml"
      ;;
    Darwin)
      ln -sf "$DOTFILES_DIR/alacritty/platform/macos.toml" "$HOME/.config/alacritty/macos.toml"
      echo "  linked: macos.toml"
      ;;
  esac
fi
```

This uses the existing `$OS` variable (set at line 18) and `$DOTFILES_DIR` (set at line 25). The `mkdir -p` ensures the directory exists even if alacritty wasn't stowed yet. Only the correct platform file is symlinked — the other platform's import in alacritty.toml will silently fail (as designed).
