Create `powershell/.config/powershell/profile.ps1` with the following content:

**Alias parity with zsh (per user decisions):**
- `Set-Alias -Name n -Value nvim` — editor
- `Set-Alias -Name dc -Value docker-compose` — docker compose
- Functions for aliases that need arguments:
  - `function gcm { git commit -m @args }` — BUT first `Remove-Alias gcm -Force -ErrorAction SilentlyContinue` because PowerShell has built-in gcm = Get-Command
  - `function gcl { git clone @args }` — git clone
  - `function t { tmux new -As workspace }` — tmux attach/create
  - `function ta { tmux attach -t workspace }` — tmux attach

**Directory shortcuts with platform branching:**
```powershell
if ($IsWindows) {
    function lab { Set-Location "$env:USERPROFILE\code\lab" }
    function proj { Set-Location "$env:USERPROFILE\code\projects" }
    function study { Set-Location "$env:USERPROFILE\code\study" }
    function cdar { Set-Location "$env:USERPROFILE\code\cdar" }
} else {
    function lab { Set-Location "$HOME/code/lab" }
    function proj { Set-Location "$HOME/code/projects" }
    function study { Set-Location "$HOME/code/study" }
    function cdar { Set-Location "$HOME/code/cdar" }
}
Set-Alias -Name s -Value study
Set-Alias -Name p -Value proj
```

**Additional sensible defaults:**
- `Set-Alias -Name python -Value python3` (with `-ErrorAction SilentlyContinue` since python3 may not exist on Windows)

Add a header comment explaining this profile mirrors the zsh/bash aliases from `~/.config/shell/config.d/alias` but is maintained independently (per user decision: standalone aliases, may drift, and that's acceptable).
