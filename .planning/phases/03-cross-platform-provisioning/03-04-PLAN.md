  <files>install.ps1</files>
  <action>
Create `install.ps1` as a standalone Windows provisioning script (no shared code with install.sh, per user decision).

**Structure:**
1. **Symlink capability check** — Test with Test-SymlinkCapability (from RESEARCH.md pattern). If symlinks fail, warn user: "Enable Developer Mode (Settings > Update & Security > For developers) or run as Administrator" and exit.

2. **Install packages via winget:**
```powershell
$packages = @(
    "Git.Git",
    "Neovim.Neovim",
    "junegunn.fzf",
    "ajeetdsouza.zoxide",
    "Alacritty.Alacritty"
)
foreach ($pkg in $packages) {
    Write-Host "Installing $pkg..."
    winget install -e --id $pkg --accept-source-agreements --accept-package-agreements
}
```
Use `--accept-source-agreements --accept-package-agreements` to avoid interactive prompts. Note: tmux is not available on Windows natively — skip it with a comment.

3. **Create symlinks** for configs that are relevant on Windows:
- nvim config: `$HOME\.config\nvim` -> `$DOTFILES_DIR\nvim\.config\nvim` (directory symlink)
- git config: `$HOME\.gitconfig` -> `$DOTFILES_DIR\git\.gitconfig`
- PowerShell profile: `$HOME\Documents\PowerShell\profile.ps1` -> `$DOTFILES_DIR\powershell\.config\powershell\profile.ps1`
- Alacritty config: `$env:APPDATA\alacritty\alacritty.toml` -> `$DOTFILES_DIR\alacritty\.config\alacritty\alacritty.toml`

Use the New-Symlink helper function pattern from RESEARCH.md (creates parent dirs, backs up existing non-symlink files).

4. **Post-install notes:**
```
Post-install steps:
  1. Open a new terminal for PATH changes to take effect
  2. Launch nvim to install plugins (first run may take a moment)
```

Keep it intentionally simple — Windows is best-effort. No error recovery, no retry logic, no WSL detection (per anti-patterns from RESEARCH.md).
