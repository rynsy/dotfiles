## Integration Checker Results

### Integration Bug

**Alacritty font size override on Linux** (APP-01, APP-02, XPLAT-01)

`alacritty.toml` unconditionally imports both `linux.toml` and `macos.toml`. Since all stow packages are deployed on all platforms, `macos.toml` overrides `font.size` from 9.0 to 13.0 on Linux. Unlike Ghostty (whose platform keys are engine-scoped), Alacritty's `font.size` is universal.

Fix options:
1. Remove `font.size` from `macos.toml`, set base to neutral value, override in `linux.toml`
2. Move all universal settings to base, use platform files only for platform-scoped keys

### DOTS-02 Mismatch

`.planning/` is tracked in git despite DOTS-02 stating ".gitignore updated to exclude .planning/". The planning files are intentionally committed — this is a documentation/requirement mismatch, not a functional issue.

### Minor Issues (Non-Breaking)

- **PATH duplication:** `zsh/.zshenv` and `shell/config.d/path` both set `$HOME/.local/bin` and `$HOME/.cargo/bin`
- **Redundant conflict check:** `install.sh` checks old `zsh/config.d/` path (harmless migration helper)
- **Hardcoded username:** `env.linux` has `/home/ryan/` in LUA_PATH/LUA_CPATH
